# WS06 Dictionary and Metadata Catalogue

## Objective and Business Outcomes
- Create a governed business dictionary and metadata catalogue for Corporate Finance Actuarial.
- Improve trust, discoverability, and consistency of reported metrics.

## In Scope
- Business glossary and data dictionary for Gold products.
- Metadata standards for table and column documentation.
- Ownership and stewardship model.

## Out of Scope
- Data quality rule implementation itself (covered in WS07).

## Inputs and Dependencies
- Inputs: WS03 target model and WS05 Gold product design.
- Dependencies: designated data owners and stewards from the business.

## Detailed Task Backlog (Sequenced)
1. Define mandatory metadata fields for all Gold assets.
2. Build dictionary entries for all critical actuarial-finance metrics.
3. Assign data owners, stewards, and technical custodians.
4. Implement catalogue quality checks (completeness and staleness).
5. Establish review and recertification cadence.

## Deliverables
- Corporate Finance Actuarial business glossary.
- Gold data dictionary.
- Ownership matrix and recertification schedule.

## RACI-lite
- Accountable: Domain data governance lead.
- Responsible: Data steward team.
- Consulted: Corporate Finance Actuarial SMEs, architecture, FE.
- Informed: wider finance and analytics consumers.

## Risks and Mitigations
- Risk: metadata quality degrades after launch.
  - Mitigation: enforce metadata completion checks in release process.
- Risk: contested ownership for cross-domain metrics.
  - Mitigation: governance forum arbitration with explicit accountability.

## Exit Criteria / Acceptance Criteria
- Metadata completeness reaches agreed threshold.
- All in-scope Gold KPIs have approved definitions and owners.

## Demo Evidence Required for Sign-off
- Live catalogue walkthrough with glossary terms, ownership, and lineage linkage.
