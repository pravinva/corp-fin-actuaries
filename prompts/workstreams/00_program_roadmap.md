# Program Roadmap: Corporate Finance Actuarial on Databricks

## Program Goal
Deliver a governed, near-real-time Corporate Finance Actuarial data and analytics platform on Databricks, replacing manual SAS-heavy processes with automated, auditable, and scalable workflows.

## Scope
- Policy, claims, reinsurance, and finance data ingestion and conformance.
- Data dictionary and metadata catalogue for Corporate Finance Actuarial.
- Automated data quality controls and live data operations.
- Portfolio monitoring dashboards, governed Genie, and anomaly monitoring.
- SAS-to-Databricks transition with dual-run and sign-off.
- Field Engineering workspace implementation package.

## 16-Week Phased Timeline

## Phase 0 (Weeks 1-2): Mobilise and Discover
- Execute `WS01 Program Governance`.
- Execute `WS02 SAS Estate Discovery`.
- Confirm stakeholders, control gates, and wave scope.

**Phase exit criteria**
- Signed governance cadence and decision rights.
- Signed SAS migration inventory with criticality and wave mapping.

## Phase 1 (Weeks 3-6): Foundation Build
- Execute `WS03 Target Data Model`.
- Execute `WS04 Data Ingestion Foundation`.
- Execute `WS06 Dictionary and Metadata Catalogue` (first iteration).

**Phase exit criteria**
- Baseline logical/physical model approved.
- Ingestion framework template operational for pilot sources.
- Mandatory metadata standard signed off.

## Phase 2 (Weeks 7-10): Core Data Products and Controls
- Execute `WS05 Conformance and Gold Products`.
- Execute `WS07 Data Quality and Controls`.
- Execute `WS08 Live Data and Pipeline Operations`.

**Phase exit criteria**
- Gold products validated against business definitions.
- DQ controls running with triage process and SLA.
- Freshness SLAs configured and monitored for pilot domains.

## Phase 3 (Weeks 11-13): Consumption and Intelligence
- Execute `WS09 Dashboard and Genie`.
- Execute `WS10 Anomaly Detection ML`.

**Phase exit criteria**
- Corporate Finance Actuarial dashboard UAT signed off.
- Genie accuracy checks completed on trusted datasets.
- Anomaly model backtesting and review workflow approved.

## Phase 4 (Weeks 14-16): Transition and Operationalisation
- Execute `WS11 SAS Transition Factory`.
- Execute `WS12 Security Governance and Lineage`.
- Execute `WS13 UAT Training and Operating Model`.
- Execute `WS14 Field Engineering Workspace Enablement`.

**Phase exit criteria**
- Dual-run reconciliations pass agreed thresholds.
- Governance and lineage controls accepted by risk/audit.
- FE deployment checklist and handover runbook approved.

## Program Controls
- Weekly delivery stand-up and RAID review.
- Fortnightly business checkpoint with Corporate Finance Actuarial.
- Monthly executive checkpoint with decision log.
- Stage-gate sign-off between phases.

## Program-Level KPIs
- % SAS workloads inventoried and wave-assigned.
- % Gold data products validated and signed off.
- DQ pass rate and mean time to resolution for critical breaches.
- Freshness SLA attainment for near-real-time feeds.
- Dashboard adoption and consumer coverage.
- % migrated outputs reconciled within tolerance.
